Detailed Description

Simple API for the communication with the Fluicell PPC1 controller.

The PPC1 controller allows the control of microfluids in the Fluicell biopen system, more details here

IMPORTANT: for the definition of all the constants, parameters and bit-conversion please refer to the documents: CommandsetManualForFluicell.pdf FluicellDeviceManager.pdf

Default parameters: Serial port baud rate 115200, 8 data bits, 1 stop bit, no parity NB! All commands must end with a newline character
Pressure setting commands. Argument f - pressure in mbar as floating point number. Vacuum channels expects the argument with minus sign.

  • "A%f\n" set vacuum on channel A
  • "B%f\n" set vacuum on channel B
  • "C%f\n" set pressure on channel C
  • "D%f\n" set pressure on channel D Example: Vacuum on channel A "A-223.4\n" or pressure on C "C402.7\n"

Valve opening or closing commands. Argument "%u" means 1 closed, 0 opened

  • "l%u\n" set valve state on channel A (Do not use in closed loop (automatic) mode)
  • "k%u\n" set valve state on channel B (Do not use in closed loop (automatic) mode)
  • "j%u\n" set valve state on channel C (Do not use in closed loop (automatic) mode)
  • "i%u\n" set valve state on channel D (Do not use in closed loop (automatic) mode)

"u%u\n" sets serial data stream period. Argument "%u" is unsigned integer marking the time period in ms. Zero turns the data stream off. (This is useful in order to catch the output of ? # or ~ commands in the terminal window) Default data stream period is 100ms, and the minimum is 25ms.
Example "u200\n" sets the data stream period to 200ms

"o%u\n" sets the TTL output state where u is either 1 or 0 (high or low)

Besides setting the state of the output with "o1\n" or "o0\n" one can generate pulses to the TTL output.

"p%u\n" generates a pulse with a length of u milliseconds. For example p30
generates a 30 milliseconds long pulse. The initial state of the output is dependent of the previous "o%u" commands (default low). If the state of output before the pulse command is high, the pulse command puts the output low for the specified amount of time. The minimum pulse length is 20 milliseconds.
"*\n" Gets the temperature of the micro-controller in degrees of Celsius. Not very accurate. "!\n" Reboots the device

Usage:

  • define the object : fluicell::PPC1api *my_ppc1 = new fluicell::PPC1api();
    • connect the device : my_ppc1->connectCOM();
    • run the thread : my_ppc1->run();
    • to stop the thread : my_ppc1->stop();
    • disconnect the device : my_ppc1->disconnectCOM();

Decode data line function.

string data format:

B|-0.000000|0.034291|0.000000|0
C|0.000000|-0.103121|0.000000|0
D|0.000000|0.028670|0.000000|0
i0|j0|k0|l0
IN1|OUT1
Current interpreted state:

Pon: set 123mbar, read: 123.456789 mbar
Poff: set 123mbar, read: 123.456789 mbar
Vswitch: set 123mbar, read: 123.456789 mbar
Vrecirc: set 123mbar, read: 123.456789 mbar
Valves: 0101

Returns
true if success, false for any error

string fluicell::PPC1api::getDeviceID ( )

Get device serial number.

Note
: from the old device api this description refers to channel, not clear: Get internal device ID. These are 6-digit serial numbers of Fluika modules in version 1 hardware and consecutive numbers in same format for version 2 hardware (e.g. "000001"). Input is channel number in range 1-6, where: 1-on pressure, 2-off pressure, 3-switch vacuum, 4-recirculation vacuum, 5-valves, 6-synchronization

Calculate the flow in a pipe using the Poiseuille equation in laminar flow, more details here

\( Q = \frac{\pi D^4 \Delta P}{128 \mu \Delta x} \)

◆ setValvesState()

bool fluicell::PPC1api::setValvesState ( const int  _value = 0x00)

Set all the valves state in one command using a binary number where 1/0 are Open/Close.

Send the string vX
set valve e...l states as one command. X is unsigned hexadecimal byte where each bit represents a valve state from MSB=e to LSB=l.

Example "vff\n" closes all valves (0xff = 0b11111111) "v0\n" opens all valves (0x00 = 0b00000000) "v0f\n" opens e to h and closes i to l (0x0f = 0b00001111) "v8a\n" closes valves e i and k, opens all others (0x8a = 0b10001010)

Parameters
_valueis an hex number
Note
- valves e to h are not supported by the PPC1
